# ctrlX CORE — Arbeitsgerät 192.168.1.1

Dieses Gerät ist das primäre Arbeitsgerät für den ctrlX Skill.

## Verbindung

| Parameter | Wert |
|---|---|
| IP-Adresse | `192.168.1.1` |
| Web UI | https://192.168.1.1 |
| REST API Base | `https://192.168.1.1/automation/api/v2` |
| Zertifikat | Self-signed → `-SkipCertificateCheck` (PS 7+) oder TrustAllCerts-Workaround (PS 5.1) |

## Standard-Credentials für den Skill (aiuser)

Für alle REST API-Aufrufe des Skills wird der dedizierte Service-Account `aiuser` verwendet.

| Parameter | Wert |
|---|---|
| Benutzername | `aiuser` |
| Passwort | Siehe `aiUserPasswort` (liegt neben `ctrlx-create-user.ps1`) |
| Scope | `rexroth-device.all.rwx` (Full access) |
| SSH | nicht erforderlich |

> Das Passwort wird nie in diesem Repository gespeichert.  
> Passwortdatei: neben dem Skript `ctrlx-create-user.ps1` unter `aiUserPasswort`.

### Token holen (PowerShell)

```powershell
$pw   = (Get-Content ".\aiUserPasswort" | Where-Object { $_ -match '^Passwort' }) -replace '.*:\s*'
$token = (Invoke-RestMethod "https://192.168.1.1/identity-manager/api/v2/auth/token" `
    -Method POST -ContentType "application/json" -SkipCertificateCheck `
    -Body (@{name="aiuser"; password=$pw} | ConvertTo-Json)).access_token
```

## Admin-Account

| Parameter | Wert |
|---|---|
| Benutzername | `boschrexroth` |
| Passwort | `boschrexroth` (Werkseinstellung) |

> Nur für administrative Aufgaben (User anlegen, Firmware-Update, …) verwenden.  
> Für normale Skill-Operationen `aiuser` bevorzugen.
